Progressive Enhancement

Progressive enhancement is a way of building websites that starts with the most basic content and features first. The idea is to make sure the website works for everyone, even if they are using an old browser or a slow internet connection. You start with a solid foundation of HTML so the text and links always work. After that, you add "layers" like CSS to make it look good and JavaScript to add cool interactive features. If someone’s browser can’t handle the fancy stuff, the website doesn't break; it just shows the simpler version that still works perfectly. This approach is really important in modern web development because people access the internet on so many different devices, from high-end computers to older smartphones. By using progressive enhancement, you ensure that your site is accessible to everyone, including people using screen readers. It makes the website more reliable and easier to maintain because you know the core of the site is strong. Instead of building something complex and trying to fix it when it breaks, you build something simple and make it better for those who have the technology to see it.

Mars Rover Reflection

This course has been an incredible journey for me. At first, I honestly thought I knew everything and expected the class to be a piece of cake. However, I ended up finding myself learning from scratch. Even though I was familiar with HTML from the bug bounty web testing I started last semester, building something from the ground up is a completely different story. I learned so much about the structure of HTML, the design power of CSS, and the logic of JavaScript. This course really gave me information I didn’t even know I needed, especially when it comes to making websites that are accessible and easy for everyone to use. When my peers tested my first draft Mars Rover site to the new one, she saw the site was very basic, with a simple background and a layout where everything was just stacked on top of each other. In the new version, I used a modern layout and a dark space theme that makes the NASA photos really stand out. I also made sure the new site is responsive, so it looks great whether you are viewing it on a phone or a computer. I’m proud of the projects I’ve finished because they challenged me to think creatively and solve real problems. In the beginning, I was just trying to get the code to work, but now I’m focused on how the user experiences the site. I’ve learned that the small details like how a button changes when you hover over it or how clear the navigation is making a huge difference. .
